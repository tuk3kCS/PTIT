Awesome! Here’s a clear, week-by-week 5-week project roadmap tailored for your Distributed File Storage System (DFS) midterm. It breaks down tasks, so you can track progress easily and hit all major milestones:

5-Week DFS Midterm Project Roadmap
Week 1: Setup & Basic File Upload/Download

Design your gRPC proto files for:

UploadFile(stream Chunk) returns UploadStatus

DownloadFile(FileRequest) returns stream Chunk

DeleteFile(FileRequest) returns DeleteStatus

Implement the master server and a single storage node

Implement a simple client to upload/download files

Test upload/download of small files locally (localhost or Docker)

Set up Docker Compose for master + storage node container

Week 2: Multiple Storage Nodes & Replication

Add support for multiple storage nodes (at least 2)

Implement replication factor = 2: master sends uploaded file chunks to two different storage nodes

Implement simple metadata tracking in master: which file is stored on which nodes (in-memory or JSON file)

Test uploading files that replicate to two nodes and downloading from either node

Week 3: Persistent Metadata & Node Status

Replace in-memory metadata with persistent metadata storage (SQLite or JSON file)

Implement a new gRPC RPC in storage nodes: GetStatus() returning disk usage and health info

Master periodically queries all storage nodes with GetStatus()

Update master metadata with node health and available free space

Test that master tracks storage node health and space correctly

Week 4: Load Balancing & Fault Tolerance

Enhance master to select least-loaded nodes for uploads based on GetStatus() data (weighted or simple least free space)

Implement delete RPC fully in master and storage nodes (remove file and update metadata)

Handle node failure:

Mark node offline if no heartbeat or no response

Uploads/downloads exclude offline nodes

Test uploading/downloading with node failures (stop a storage node mid-demo)

Week 5: Polish & Optional Auto-Scaling Simulation

Add heartbeat mechanism: storage nodes ping master regularly to confirm liveness

(Optional) Implement a simple auto-scaling simulation:

Master spawns new Docker storage node containers when average free space < threshold

Master can shut down idle nodes gracefully

Write a final report summarizing your design, protocol, and testing results

Prepare a demo script showing uploads, replication, node failures, and recovery

Bonus Tips:

Keep your proto files clean and versioned.

Write reusable client scripts for uploading/downloading.

Use Docker volumes to persist node storage between restarts.

Log key events (uploads, deletes, node status) to console for easy demo.

Consider adding file checksum verification (MD5/SHA256) for integrity.
